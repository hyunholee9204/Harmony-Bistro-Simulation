# 🍽 Harmony Bistro Simulation

## 📌 프로젝트 소개
**Harmony Bistro Simulation**은 레스토랑 웹 페이지를 직접 구현하고,  
그 위에 사용자 행동에 따라 시스템이 반응하는 인터랙티브 요소를 결합한 프로젝트입니다.
 
손님 입장 → 주문 → 주방 조리 → 완료 → 결제까지 이어지는 흐름을  
상태 기반 로직으로 설계하여 실제 레스토랑 운영 과정을 시뮬레이션 형태로 구현했습니다.

특히 사용자 클릭과 같은 입력에 따라 테이블 상태, 주문 진행 상황, 결제 가능 여부가  
동적으로 변화하도록 구성하여, **웹 UI와 로직이 유기적으로 연결된 인터랙티브 시스템**을 구현하는 데 초점을 맞췄습니다.
---

## 🎮 주요 기능

### 👥 테이블 및 손님 관리
- 랜덤 인원의 손님 입장
- 대기열 기반 테이블 배정 시스템
- 테이블 상태 변화  
  `EMPTY → OCCUPIED → COOKING → READY`

### 📝 주문 시스템
- 메뉴 카테고리별 랜덤 주문 생성
- 주문 수량은 인원 기반으로 동적 결정
- 주문 내역 확인 (영수증 UI 제공)

### 👨‍🍳 주방 조리 시스템
- 한식 / 중식 / 양식 주방 분리
- 주문 메뉴 자동 분배
- 조리 완료 버튼을 통한 개별 처리

### ⏱ 조리 완료 로직
- 메뉴 단위 완료 처리
- 모든 메뉴 완료 시에만 결제 가능
- 부분 완료 상태에서는 결제 제한

### 💳 결제 시스템
- 계산대에서 테이블별 결제 진행
- 총 금액 + 팁 계산 반영
- 결제 완료 시 테이블 상태 초기화

### 🧾 영수증 UI
- 메뉴별 가격 표시
- 총 금액 계산
- 영수증 스타일 UI 적용

## 🧠 시스템 설계 포인트

### ✔ 상태 기반 설계 (State Management)

```
EMPTY → OCCUPIED → COOKING → READY → EMPTY
```

- 상태에 따라 가능한 행동을 제한
- 비정상적인 흐름 방지

---

### ✔ 이벤트 기반 로직

- 행동 → 상태 변화 → UI 반영 구조

---

- 유지보수 및 확장성 고려
